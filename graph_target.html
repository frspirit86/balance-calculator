변경 1 Weird 존 타겟 후보군 완전 제외
이유: Weird 존(공이 엉뚱하게 튀는 경우)은 타겟 선정 이벤트가 아닌 별도 이벤트로 처리하는 것이 적합하다. 기존에는 Weird_L / Weird_R 두 항목이 softmax 후보군에 포함되어, 수비 능력치가 낮아 α가 낮아질수록 Weird 선택 확률이 과도하게 높아지는 문제가 있었다.
v1 (Weird 포함)
ZONES = [
  Normal_Long,
  Normal_Short,
  Weird_L,   // softmax 풀 참여
  Weird_R,   // softmax 풀 참여
]
최종 (Weird 제외)
ZONES = [
  Normal_Long,
  Normal_Short,
  // Weird → 별도 이벤트로 분리
]
변경 2 확률 배분 공식 — 순위 퍼센타일 → √ 압축 Softmax
이유: 퍼센타일 방식은 후보 수가 적을 때(2명이면 항상 {1.0, 0.0}) 점수 차이가 무시되는 문제가 있었다. Raw softmax는 반대로 점수 차이가 클 때 1위가 확률을 독식하는 문제가 있다. √ 압축 softmax는 두 문제를 모두 완화한다.
v1 (순위 퍼센타일)
p_i = (N - rank_i) / (N - 1)
→ softmax(α × p_i)
// 2명이면 항상 {1.0, 0.0}
최종 (√ 압축 Softmax)
c_i = √(score_i - score_min)
→ softmax(α × c_i)
// 점수 차이 비례, N 무관
구현 코드
function calcProbs(rankedCandidates, alpha) {
  const sMin = Math.min(...rankedCandidates.map(c => c.totalScore));
  const result = rankedCandidates.map(c => {
    const compressed = Math.sqrt(Math.max(0, c.totalScore - sMin));
    return { ...c, expVal: Math.exp(alpha * compressed) };
  });
  const sumExp = result.reduce((s, c) => s + c.expVal, 0);
  return result.map(c => ({ ...c, prob: c.expVal / sumExp }));
}
변경 3 α(집중도) 하한값 추가 — alphaMin
이유: 수비 능력치가 낮거나 상대 공격자 능력치가 높아 α가 0에 수렴하면 softmax 집중 효과가 소실되어 모든 후보가 동일 확률이 되고 점수 계산이 무의미해진다. alphaMin으로 최소 집중도를 보장한다 (기본값 0.3).
구현 코드
// α 계산 후 하한 적용
return Math.max(alphaMin, a);  // alphaMin 기본값: 0.3
변경 4 상대 진영 아군 타겟 점수 감산 — Score_UpperSide_DefenceEventTarget
이유: 상대 진영(하프라인 위)에 위치한 아군 타겟은 공격 포지셔닝 상태이므로, 태클 후 해당 아군에게 공이 전달되면 즉각 공격 기회가 된다. 이를 보정하기 위해 하프라인을 넘어선 아군 타겟에 한해 위치 점수를 일괄 감산한다. 하프라인 이하(수비 진영)에는 적용되지 않는다.
적용 전
// 진영 무관하게 동일 점수
아군_점수 = (N − rank_i)
// 상대 진영 아군도 동일 취급
적용 후
if (아군.x > 0) {  // 하프라인 위
  점수 += Score_UpperSide_DefenceEventTarget
  // 기본값 -2 (감산)
}
파라미터
Score_UpperSide_DefenceEventTarget — 기본값 -2. 음수일수록 상대 진영 아군 타겟을 강하게 억제. 0이면 기존 동작과 동일.
